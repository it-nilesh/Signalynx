# Security Policy

## Supported Versions

Security fixes are provided for the latest stable release. Pre-release builds
and older versions may receive fixes at the maintainers' discretion.

## Reporting a Vulnerability

Do not open a public issue for a suspected vulnerability. Report it privately
through the repository host's security-advisory feature or the private contact
method published by the maintainers.

Include:

- affected package and version;
- a minimal reproduction;
- expected and actual behavior;
- potential impact;
- any suggested mitigation.

Maintainers should acknowledge a complete report within five business days.
No response-time or remediation warranty is provided.

Signalynx is an in-process dispatch library. Applications remain responsible
for authorization, input validation, data protection, secret management,
transport security, audit requirements, and regulatory compliance.
